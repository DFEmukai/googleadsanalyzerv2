## 1. はじめに

### 1.1 Google Ads AI Agentとは

Google Ads AI Agentは、Google広告アカウントの運用を支援するAIツールです。以下の機能を提供します：

- **自動分析**: 週次でGoogle広告のパフォーマンスを分析
- **改善提案**: AIが具体的な改善アクションを提案
- **承認ワークフロー**: 提案を確認・承認・実行
- **効果測定**: 施策実行後の効果を自動計測

### 1.2 アクセス方法

ブラウザで以下のURLにアクセスします：
https://frontend-qja7.onrender.com/dashboard

ログイン認証は現在実装されていません（社内利用想定）。

## 2. ダッシュボードの見方

### 2.1 ダッシュボード画面

サイドバーの「ダッシュボード」をクリックするか、トップページにアクセスするとダッシュボードが表示されます。

### 2.2 KPIサマリーカード

画面上部に表示される6つのカードで、**直近7日間**の主要KPIを確認できます。

| KPI | 説明 | 計算方法 |
|-----|------|----------|
| 費用 | 広告費用の合計 | 7日間の合計 |
| CV数 | コンバージョン数 | 7日間の合計 |
| CPA | 獲得単価 | 総費用 ÷ 総CV数 |
| CTR | クリック率 | 総クリック ÷ 総インプレッション × 100 |
| ROAS | 広告費用対効果 | 総CV価値 ÷ 総費用 |
| インプレッション | 表示回数 | 7日間の合計 |

### 2.3 シグナル（色）の意味

各KPIカードの色は、前週との比較に基づいて表示されます：

| 色 | 意味 | 説明 |
|----|------|------|
| 🟢 緑 | 良好 | 前週より改善 |
| 🟡 黄 | 注意 | 前週と同程度 |
| 🔴 赤 | 要対応 | 前週より悪化 |
| 🔵 青 | 情報 | 中立的な指標（費用、インプレッションなど） |

### 2.4 前週比較

各KPIカードには以下の情報が表示されます：

- **現在値**: 直近7日間の値
- **前週値**: 8日前〜14日前の値
- **変化率**: （現在値 - 前週値）÷ 前週値 × 100%
- **矢印**: ↑（増加）または ↓（減少）

### 2.5 トレンドグラフ

画面下部には、**直近7日間の日別推移**がグラフで表示されます：

- **X軸**: 日付（例: 02-13, 02-14, 02-15...）
- **Y軸**: 各KPIの値
- **グラフ種類**: CPA推移、CV数推移、費用推移、ROAS推移

### 2.6 保留中の提案

右上に「保留中の提案: X件」と表示されます。クリックすると改善提案ページに遷移します。

## 3. キャンペーン一覧・詳細の使い方

### 3.1 キャンペーン一覧画面

サイドバーの「キャンペーン」をクリックすると、キャンペーン一覧が表示されます。

#### 表示項目

| 項目 | 説明 |
|------|------|
| ステータス | 🟢 有効 / 🟡 停止中 / ⚫ 終了 |
| キャンペーン名 | Google広告のキャンペーン名 |
| タイプ | search / display / pmax / video |
| 費用 | 期間内の広告費用 |
| CV | コンバージョン数 |
| CPA | 獲得単価 |
| CTR | クリック率 |
| ROAS | 広告費用対効果 |

#### ソート機能

テーブルのヘッダーをクリックすると、その項目でソートできます：

- キャンペーン名（あいうえお順/逆順）
- 費用（高い順/低い順）
- CV数（多い順/少ない順）
- CPA（高い順/低い順）
- CTR（高い順/低い順）
- ROAS（高い順/低い順）

### 3.2 キャンペーン詳細画面

一覧でキャンペーン行をクリックすると、詳細ダッシュボードが表示されます。

#### KPIカード

そのキャンペーンの直近7日間のKPIが表示されます：

- 費用
- CV数
- CPA
- CTR
- ROAS
- インプレッション

#### 日別トレンドグラフ

期間内の日別パフォーマンス推移がグラフで表示されます。

#### 期間選択

画面上部のドロップダウンで表示期間を変更できます：

- 7日間
- 14日間
- 30日間

#### 関連提案

このキャンペーンに対する改善提案がある場合、画面下部に一覧表示されます。

## 4. 改善提案の確認・壁打ち・承認方法

### 4.1 改善提案一覧画面

サイドバーの「提案」をクリックすると、改善提案一覧が表示されます。

#### フィルター機能

画面上部のフィルターで提案を絞り込めます：

| フィルター | 選択肢 |
|------------|--------|
| ステータス | すべて / 承認待ち / 承認済み / 実行済み / 却下 / スキップ |
| カテゴリ | すべて / キーワード / クリエイティブ / 手動クリエイティブ / ターゲティング / 予算 / 入札 / 競合対応 |
| 優先度 | すべて / 高 / 中 / 低 |

#### 提案カードの見方

各提案カードには以下の情報が表示されます：

- **優先度バッジ**: 🔴高 / 🟡中 / 🟢低
- **カテゴリバッジ**: keyword / creative / budget など
- **タイトル**: 提案の概要
- **対象**: キャンペーン名・広告グループ名
- **期待効果**: 実行した場合の予測効果

### 4.2 提案の詳細確認

提案カードをクリックすると、詳細が展開されます：

- **説明**: 提案の詳細な説明
- **実行ステップ**: 具体的なアクション手順
- **ステータス**: 現在のステータス

### 4.3 壁打ちチャット機能

提案について疑問がある場合、AIと対話して検討できます。

#### 使い方

1. 提案カードを展開
2. 「Claudeと壁打ち」ボタンをクリック
3. チャットウィンドウが表示される
4. 質問やコメントを入力して送信

#### 使用例

```
あなた: この提案の根拠をもう少し詳しく教えてください
Claude: この提案は、過去2週間のキーワードパフォーマンスデータに基づいています...

あなた: 代替案はありますか？
Claude: はい、以下の代替案が考えられます...

あなた: リスクはありますか？
Claude: 想定されるリスクとしては...
```

### 4.4 提案の承認方法

#### 承認待ち（pending）の提案に対するアクション

| ボタン | 動作 |
|--------|------|
| **承認して実行** | 提案を承認し、即座にGoogle Adsに反映 |
| **承認のみ** | 提案を承認（後で手動実行） |
| **却下** | 提案を却下（理由入力可） |
| **スキップ** | 一時的に保留（後で再検討） |

#### 承認して実行の手順

1. 「承認して実行」ボタンをクリック
2. 確認ダイアログが表示される
3. 実行者名を入力（任意）
4. 「実行」ボタンをクリック
5. Google Ads APIを通じて変更が反映される
6. ステータスが「実行済み」に変更

#### 提案内容の編集

承認前に提案内容を編集できます：

1. 「編集して承認」をクリック
2. 変更したい項目を編集
3. 「承認」ボタンをクリック

### 4.5 承認済み提案の実行

「承認のみ」で承認した提案は、後から実行できます：

1. ステータスフィルターで「承認済み」を選択
2. 対象の提案を展開
3. 「実行」ボタンをクリック

### 4.6 実行済み提案のロールバック

実行後24時間以内であれば、変更を元に戻せます：

1. ステータスフィルターで「実行済み」を選択
2. 対象の提案を展開
3. 「ロールバック」ボタンをクリック
4. ロールバック理由を入力
5. 「確定」ボタンをクリック

**注意**: ロールバック可能なのは実行から24時間以内のみです。

### 4.7 手動クリエイティブ提案

「手動クリエイティブ」カテゴリの提案は、画像や動画の作成が必要なため自動実行できません。

- Chatworkタスクとして担当者に自動登録されます
- 提案カードに「手動対応」バッジが表示されます

## 5. 週次レポートの見方

### 5.1 レポート一覧画面

サイドバーの「レポート」をクリックすると、週次レポート一覧が表示されます。

#### 表示項目

| 項目 | 説明 |
|------|------|
| 期間 | レポート対象週（例: 2024/02/12 - 2024/02/18） |
| 作成日 | レポートが生成された日時 |
| 提案数 | このレポートに含まれる改善提案の数 |
| 費用 | 週間の総費用 |
| CV数 | 週間のコンバージョン数 |
| CPA | 週間のCPA |

### 5.2 レポート詳細画面

レポート行をクリックすると、詳細が表示されます。

#### 分析サマリー

AIによる週次パフォーマンスの分析結果が表示されます：

- 全体的なパフォーマンス評価
- 注目すべきトレンド
- 主要な課題と機会

#### KPIスナップショット

週間の主要KPIがカードで表示されます。

#### 改善提案一覧

このレポートから生成された改善提案が一覧表示されます。カテゴリ・優先度でフィルタリング可能です。

## 6. 分析実行ボタンの使い方

### 6.1 手動分析の実行

通常、分析は毎週月曜7:00に自動実行されますが、任意のタイミングで手動実行することもできます。

#### 実行方法

1. ダッシュボードまたはレポート画面を開く
2. 「分析を実行」ボタンをクリック
3. 確認ダイアログが表示される
4. 「実行」ボタンをクリック
5. 分析が開始される（数分かかる場合があります）
6. 完了すると新しいレポートが作成される

### 6.2 期間指定での分析

デフォルトでは前週（月曜〜日曜）のデータを分析しますが、任意の期間を指定することもできます：

1. 「分析を実行」ボタンをクリック
2. 「期間を指定」オプションを選択
3. 開始日と終了日を入力
4. 「実行」ボタンをクリック

### 6.3 分析の所要時間

分析には以下の処理が含まれるため、完了まで数分かかります：

1. Google Ads APIからデータ取得
2. Claude AIによる分析
3. 改善提案の生成
4. データベースへの保存
5. Chatwork通知（設定時）

## 7. 効果レポートの見方

### 7.1 効果レポートとは

実行した改善提案の効果を測定するレポートです。実行前後のKPIを比較して、施策の効果を可視化します。

### 7.2 効果レポートの確認方法

1. 改善提案一覧で「実行済み」フィルターを選択
2. 効果を確認したい提案を展開
3. 「効果レポート」ボタンをクリック

### 7.3 効果レポートの内容

#### 比較期間

| 期間 | 説明 |
|------|------|
| Before（実行前） | 提案実行前の7日間 |
| After（実行後） | 提案実行後の7日間 |

#### 比較項目

| KPI | 表示内容 |
|-----|----------|
| 費用 | 実行前の値 → 実行後の値（変化率%） |
| CV数 | 実行前の値 → 実行後の値（変化率%） |
| CPA | 実行前の値 → 実行後の値（変化率%） |
| CTR | 実行前の値 → 実行後の値（変化率%） |
| ROAS | 実行前の値 → 実行後の値（変化率%） |
| インプレッション | 実行前の値 → 実行後の値（変化率%） |
| クリック | 実行前の値 → 実行後の値（変化率%） |
| CV価値 | 実行前の値 → 実行後の値（変化率%） |

#### 変化率の見方

- **プラス（＋）**: 数値が増加
- **マイナス（－）**: 数値が減少
- **緑色**: 改善方向への変化（例: CPAが減少、ROASが増加）
- **赤色**: 悪化方向への変化（例: CPAが増加、ROASが減少）

### 7.4 注意事項

- 効果測定には、実行後7日間のデータが必要です
- 実行直後は「効果レポート」が表示されない場合があります
- 外部要因（季節性、競合変動など）の影響も含まれる可能性があります

## よくある質問（FAQ）

### Q1: データが更新されていないようです

A: Google Ads APIのデータは最大24時間の遅延があります。また、当日分のデータは不完全な場合があります。翌日に再度確認してください。

### Q2: 提案を間違えて実行してしまいました

A: 実行から24時間以内であれば、「ロールバック」機能で元に戻せます。24時間を過ぎた場合は、Google広告管理画面から手動で修正してください。

### Q3: 壁打ちチャットの履歴は保存されますか？

A: はい、提案ごとにチャット履歴が保存されます。次回アクセス時も過去の会話を確認できます。

### Q4: 週次レポートが生成されていません

A: 以下を確認してください：
- Google Ads APIの認証情報が正しく設定されているか
- Anthropic APIキーが有効か
- サーバーが正常に稼働しているか

### Q5: 手動クリエイティブ提案はどう対応すればよいですか？

A: 手動クリエイティブ提案は、画像や動画の作成が必要なため自動実行できません。Chatworkタスクとして登録されるので、担当者が手動で対応してください。
